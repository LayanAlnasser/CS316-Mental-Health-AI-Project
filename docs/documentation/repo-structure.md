# Repository Structure

The repository is organized to separate documentation, methodology, results, datasets, and deployment components in a clear and scalable manner.

## Directory Overview

| Path | Description |
|-----|------------|
| `docs/index.md` | Project landing page |
| `docs/introduction.md` | Project background and motivation |
| `docs/objectives.md` | Project goals and scope |
| `docs/dataset.md` | Dataset overview and access |
| `docs/ethics.md` | Ethical and societal considerations |
| `docs/datasets/` | Raw dataset files (Excel / PDF) |
| `docs/methodology/data.md` | Data generation and preprocessing |
| `docs/methodology/models.md` | Machine learning models and algorithms |
| `docs/methodology/evaluation.md` | Evaluation metrics and methodology |
| `docs/results/findings.md` | Experimental results and analysis |
| `docs/results/error-analysis.md` | Error patterns and model behavior |
| `docs/results/limitations.md` | Technical and practical limitations |
| `docs/deployment/app.md` | Application overview |
| `docs/deployment/api.md` | API usage and endpoints |
| `docs/deployment/install.md` | Setup and installation instructions |
| `docs/paper/paper.md` | IEEE-style research paper |
| `docs/paper/slides.md` | Presentation slides |
| `docs/paper/artifacts.md` | Supplementary materials |
| `docs/documentation/ethics.md` | Ethics and responsible AI discussion |
| `docs/documentation/future-work.md` | Future research directions |
| `docs/documentation/repo-structure.md` | Repository organization reference |
| `docs/documentation/requirements.md` | Software and hardware requirements |

---

## Design Rationale

- Separates **research methodology**, **experimental results**, and **deployment**
- Improves navigation and readability for reviewers and graders
- Supports scalability and reproducibility
- Aligns with academic and software engineering best practices
